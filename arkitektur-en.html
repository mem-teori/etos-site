M.E.M. / ETOS is not one system — it is nine interconnected modules that together can identify, structure, analyse and learn from complex decision situations.

Layer 1 · Theoretical foundation

M.E.M. — Meaning, Experience, Model

The conceptual foundation that describes decision situations as relationships between three layers. Status: Conceptual model ready for empirical testing.

🔬
Three dimensions — one tension
Next: empirical testing in organisational studies
M

Meaning

Normative values the system seeks to protect — safety, autonomy, dignity and fairness

E

Experience

Human experience — moral stress, loss of control, uncertainty and trust in the system

M

Model

Institutional structures — laws, procedures, algorithms and decision frameworks

Layer 2 · Decision engine

IDK — Decision Engine

Structures decisions in situations with normative tension. Classifies into five states based on six parameters. Also calculates ethical debt as a function of divergence. Status: Working prototype — tested in 1,296 synthetic cases.

Normal
The decision is in balance
Needs review
Emerging tension — review recommended
Emergency
High tension — ethical debt is registered
Incompatible
The system cannot fulfil its own goals
Constitutional veto
Foundational principles are broken — requires human intervention

Assesses via these parameters:

Coercion Refusal Capacity loss Risk Time sensitivity Advance directive

Ethical debt

Calculated as divergence × 0.85. Classified as Low / Moderatee / High / Critical.

Test result

1,296 synthetic cases · 23% classified as Emergency · Clear structural patterns.

Layer 3 · Software and analysis

ETOS — The software architecture and its modules

Six modules that implement M.E.M. and IDK in a software environment — from data registration to pattern analysis and visualisation.

ETOS Core System Prototype

Registers decision parameters, classifies situations, logs decisions and analyses patterns over time.

Next: stable user interface + data security
The Reflection Centre Fungerende

Collects cases, identifies patterns and makes systemic tensions visible. Shifts focus from individual errors to structural conditions.

Next: organisation reports + visualisations
Scenario Simulator Forskningsværktøj

Generates hypothetical cases and tests the system's logic. Has run 1,296 synthetic scenarios with clear structural patterns.

Next: realistic data + sector-specific scenarios
Weakpoint Analyzer Experimental

Finds systemic weaknesses in the decision engine. Has identified 22 cases of "low score emergency" and 12 "abrupt transitions".

Next: automatic suggestions for improvements
Decision Landscape Experimental

Maps how the system responds to different situation types — visualises the decision pattern across parameters.

Next: interactive visualisations + policy analysis
Learning Loop Early prototype

Lets the system learn from previous decisions — identifies recurring patterns and suggests adjustments to the decision logic.

Next: adaptive parameter adjustment
Layer 4 · Governance

CDP — Constitutional framework

CDP ensures the decision system remains transparent, auditable and institutionally accountable. A constitutional layer above the other modules. Status: Konceptuel model.

CDP fungerer som et rammeværk der stiller krav til alle øvrige lag — ikke som et modul der løser specifikke opgaver, men som den institutionelle garanti for at systemet kan bruges ansvarligt.

👁 Transparens — beslutningslogikken er synlig og forståelig
📋 Auditabilitet — alle beslutninger kan efterprøves
🏛 Institutionelt ansvar — systemet erstatter ikke peoplelig vurdering

"A system that makes decisions about people must itself be accountable — not hide behind the logic of the algorithm."
